# 🧪 R&D Brain - 研究開発アイデア創出AIエージェント

> **化学メーカーの研究開発を加速するAIエージェント**

営業担当者が顧客との面談内容を入力するだけで、AIが自動的に内容を分析し、社内の他事業部の知見や市場トレンドを統合して、**新規用途や改良アイデアを提案する戦略レポート**を自動生成します。

## 🎯 このアプリで解決すること

### 課題
- 営業担当者が顧客から聞いた技術的なニーズが、社内の他事業部の知見と結びつかない
- 面談録がバラバラに保存され、過去の知見を活かしきれない
- 市場トレンドや新技術の情報を手動で収集するのが大変

### 解決策
- **AIが自動的に面談内容を評価**し、化学的な具体的なニーズを抽出
- **ベクトル検索で類似知見を自動発見**（他事業部の知見とのクロスポリネーション）
- **市場トレンドを自動調査**し、戦略レポートとして統合

## 💡 使用例

### シナリオ
営業担当者が自動車メーカーと面談し、「車体の軽量化のために、100℃以上の高温でも変形しない樹脂材料を探している」というニーズを聞いた場合。

### アプリでの処理
1. **入力**: 面談内容をテキストで入力
2. **AIレビュー**: 「高温耐性」「樹脂材料」「軽量化」などの技術タグを自動抽出
3. **保存**: ベクトル化してデータベースに保存
4. **自動探索**: 
   - 他事業部の類似知見（例：エバール事業部が以前に高温耐性材料で対応した事例）
   - 市場トレンド（例：自動車向け軽量材料の最新動向、2024年の規制動向）
5. **レポート生成**: 「車体用高温耐性樹脂の新規用途提案」として統合レポートを生成

### 結果
- 他事業部の知見を活用した提案が可能に
- 市場トレンドを踏まえた戦略的な提案が可能に
- 営業担当者が一人で完結できる情報収集・分析が実現

## 🚀 主な機能

### 1. 面談録の入力とAIレビュー
- 営業担当者が面談内容を入力
- GPT-4oが内容を自動評価
- 化学的な具体的なニーズ（温度、強度、耐性など）が含まれているかを判定
- 情報不足の場合は追加質問を提示

### 2. データベースへの保存
- 面談内容をEmbedding化してSupabaseに保存
- メタデータ（企業名、部署、事業部、技術タグ、登録日時）を付与

### 3. アイデア創出プロセス（自動実行）
データ登録完了後に自動で以下を実行：

#### 3.1 社内シーズの探索
- 他事業部の類似知見をベクトル検索で探索
- 現在の事業部と異なるデータのみを取得（クロスポリネーション）

#### 3.2 市場調査
- DuckDuckGo検索を使用して最新の市場トレンドを調査
- 技術タグと用途から関連する規制や新技術を検索

#### 3.3 戦略レポート生成
以下のセクションを含むMarkdownレポートを自動生成：
- **Trigger**: 今回の顧客の声（企業名・ニーズ）
- **Chemical Insight**: 抽出された化学的課題
- **Cross-Link**: 社内の他事業部にある類似知見（関連度とその理由）
- **Market Trend**: 関連する市場の動き
- **Proposal**: クラレとして提案すべき「新用途」または「改良アイデア」

## 📦 セットアップ

### 前提条件

このアプリを動かすために必要なもの：

- **Python 3.12以上**
  - 確認方法: `python3 --version` または `python --version`
  - インストール: [python.org](https://www.python.org/downloads/) からダウンロード
  
- **Supabaseアカウントとプロジェクト**
  - アカウント作成: [supabase.com](https://supabase.com) で無料アカウントを作成
  - プロジェクト作成: ダッシュボードから新しいプロジェクトを作成
  
- **OpenAI APIキー**
  - アカウント作成: [platform.openai.com](https://platform.openai.com/) でアカウントを作成
  - APIキー取得: API Keysセクションから新しいキーを作成（有料プランが必要）

### インストール手順（初回セットアップ）

#### ステップ1: リポジトリのクローン（またはダウンロード）

```bash
# Gitを使用する場合
git clone <repository-url>
cd R-D_AIagent_MVP

# または、ZIPファイルをダウンロードして解凍した場合
cd R-D_AIagent_MVP
```

#### ステップ2: 仮想環境の作成と有効化

**macOS/Linuxの場合:**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windowsの場合:**
```bash
python -m venv venv
venv\Scripts\activate
```

**確認**: コマンドプロンプトの先頭に `(venv)` が表示されれば成功です。

#### ステップ3: 依存パッケージのインストール

```bash
# pipを最新版にアップグレード
pip install --upgrade pip

# 必要なパッケージをインストール
pip install -r requirements.txt
```

**インストールされる主要パッケージ:**
- `streamlit` - Webアプリケーションフレームワーク
- `langchain` - LLM統合フレームワーク
- `openai` - OpenAI APIクライアント
- `supabase` - Supabaseクライアント
- `duckduckgo-search` - DuckDuckGo検索API

#### ステップ4: 環境変数の設定

1. **`.env`ファイルを作成**

   ```bash
   # env.exampleをコピーして.envファイルを作成
   cp env.example .env
   ```

   **Windowsの場合:**
   ```cmd
   copy env.example .env
   ```

2. **`.env`ファイルを編集**
   
   エディタで`.env`ファイルを開き、実際の値を設定します：

   ```bash
   SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co
   SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   OPENAI_API_KEY=sk-...
   ```

   **各値の取得方法:**
   - **SUPABASE_URL**: Supabaseダッシュボード > Settings > API > Project URL
   - **SUPABASE_KEY**: Supabaseダッシュボード > Settings > API > anon public キー
   - **OPENAI_API_KEY**: OpenAI Platform > API Keys > 新しいキーを作成

   詳細は [環境変数の設定](#環境変数の設定) を参照してください。

#### ステップ5: Supabaseデータベースの設定

1. **Supabaseダッシュボードにアクセス**
   - [supabase.com](https://supabase.com) にログイン
   - 作成したプロジェクトを選択

2. **SQL Editorを開く**
   - 左サイドバーの「SQL Editor」をクリック

3. **SQLを実行**
   - `supabase_settings.txt` ファイルの内容をコピー
   - SQL Editorに貼り付けて実行（「Run」ボタンをクリック）
   
   これにより、以下のものが作成されます：
   - `documents`テーブル（面談録を保存）
   - `match_documents`関数（ベクトル検索用）

4. **確認**
   - 左サイドバーの「Table Editor」を開く
   - `documents`テーブルが作成されていることを確認

### セットアップの確認

すべての設定が完了したら、アプリを起動して確認します：

```bash
# 仮想環境が有効になっていることを確認（先頭に(venv)が表示される）
streamlit run app.py
```

ブラウザで `http://localhost:8501` が開き、サイドバーに「✅ すべてのAPIキーが設定されています」と表示されれば成功です。

- 仮想環境が有効になっているか確認

### 基本的な使い方（ステップバイステップ）

#### ステップ1: 事業部の選択
- 画面左側のサイドバーから事業部を選択します
- 選択可能な事業部：
  - エバール事業部
  - イソプレン事業部
  - ジェネスタ事業部
- **重要**: 選択した事業部以外の知見が検索対象になります（クロスポリネーション）

#### ステップ2: 面談情報の入力
フォームに以下の情報を入力します：

- **企業名 (Company Name)**
  - 例: `トヨタ自動車株式会社`
  
- **相手方 部署・役職**
  - 例: `ボディ設計部 課長`
  
- **面談メモ (Raw Content)**
  - 面談の内容を自由に記述してください
  - 以下の情報が含まれていると、より精度の高い分析が可能です：
    - 具体的な数値（温度、強度、サイズなど）
    - 現行品の問題点
    - 求められる性能・特性
    - 用途・アプリケーション

**入力例**:
```
顧客は自動車の軽量化を目的として、新しい樹脂材料を探しています。
特に、100℃以上の高温環境でも変形せず、かつ引張強度が100MPa以上の
材料を希望しています。現在使用している材料は、高温時に軟化してしまう
問題があります。
```

#### ステップ3: AIレビュー実行
- 「**AIレビュー実行**」ボタンをクリックします
- AI（GPT-4o）が面談内容を評価し、以下の結果を表示します：

**✅ 情報が十分な場合**:
- 内容要約が表示されます
- 抽出された技術タグが表示されます（例: `耐熱性`, `樹脂材料`, `軽量化`）

**⚠️ 情報が不足している場合**:
- 追加で確認すべき質問が表示されます
- 具体的な数値や現行品の問題点などを追加で記入するよう促されます

#### ステップ4: データの登録
- AIレビューで「情報が十分」と判定された場合のみ、「**この内容で登録しますか？**」ボタンが表示されます
- ボタンをクリックすると：
  1. 面談内容がベクトル化（Embedding化）されます
  2. Supabaseデータベースに保存されます
  3. メタデータ（企業名、事業部、技術タグ、登録日時）も同時に保存されます

#### ステップ5: アイデア創出レポートの確認
データ登録完了後、自動的に以下のプロセスが実行されます：

1. **🔍 社内の他事業部の知見を探索中...**
   - ベクトル検索で類似する過去の面談録を探索
   - 現在の事業部と異なるデータのみを取得

2. **🌐 市場トレンドを調査中...**
   - DuckDuckGo検索で最新の市場情報を収集
   - 技術タグと用途から関連する規制や新技術を検索

3. **📊 戦略レポートを生成中...**
   - 収集した情報を統合してMarkdown形式のレポートを生成

**レポートには以下が含まれます**:
- **Trigger**: 今回の顧客の声（企業名・ニーズ）
- **Chemical Insight**: 抽出された化学的課題
- **Cross-Link**: 社内の他事業部にある類似知見（関連度とその理由）
- **Market Trend**: 関連する市場の動き
- **Proposal**: クラレとして提案すべき「新用途」または「改良アイデア」

さらに、**他事業部の類似知見**がカード形式で表示され、関連度が高い順に並びます。

### よくある質問（FAQ）

**Q: 情報が不足していると判定された場合、どうすればよいですか？**
A: 表示された追加質問に答える形で、面談メモを更新してから再度「AIレビュー実行」をクリックしてください。

**Q: レポートは保存されますか？**
A: 現在のバージョンでは、レポートは表示のみでデータベースには保存されません。必要に応じて手動でコピーしてください。

**Q: 過去の面談録を検索できますか？**
A: 現在のバージョンでは、新規登録時の自動検索のみです。過去のレポート一覧機能は今後の拡張予定です。

### 主要ファイルの詳細

#### `app.py` - メインアプリケーション
- **役割**: Streamlitアプリケーションのエントリーポイント
- **主な機能**:
  - 事業部選択UI（サイドバー）
  - 面談情報入力フォーム
  - AIレビュー実行（`review_interview_content()`関数）
  - 戦略レポート生成（`generate_idea_report()`関数）
  - レポート表示と他事業部知見のカード表示

#### `backend.py` - バックエンド処理
- **役割**: データベース操作と外部API連携
- **主な関数**:
  - `save_interview_note()` - 面談録をベクトル化してSupabaseに保存
  - `search_cross_pollination()` - 他事業部の類似知見をベクトル検索
  - `search_market_trends()` - DuckDuckGoで市場トレンドを検索

#### `services/` - 拡張用サービスモジュール
- **現状**: ファイルは存在するが、現在のMVPでは未使用
- **将来の拡張**: 学術論文、ニュース、特許情報などの検索機能を追加可能

#### `supabase_settings.txt` - データベース設定
- **内容**: PostgreSQL + pgvector拡張を使用したデータベース設定
- **重要な要素**:
  - `documents`テーブル（id, content, metadata, embedding）
  - `match_documents`関数（ベクトル類似度検索）

## 🔐 環境変数の設定

### 環境変数の設定方法

プロジェクトルートに `.env` ファイルを作成し、以下の形式で設定してください：

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-key
OPENAI_API_KEY=sk-...
```

`.env` ファイルは自動的に読み込まれます（`python-dotenv`を使用）。

### 環境変数の取得方法

#### Supabase
1. Supabaseプロジェクトのダッシュボードにアクセス
2. Settings > API から以下を取得：
   - **URL**: Project URL
   - **Key**: `anon` `public` キー

#### OpenAI
1. [OpenAI Platform](https://platform.openai.com/) にアクセス
2. API Keys から新しいキーを作成または既存のキーを取得

## 🗄️ Supabase設定

### データベーステーブル

`documents` テーブルが必要です。以下の構造で作成してください：

```sql
CREATE TABLE documents (
  id BIGSERIAL PRIMARY KEY,
  content TEXT NOT NULL,
  metadata JSONB,
  embedding vector(1536)
);
```

### 検索関数

`supabase_settings.txt` の内容をSupabaseのSQL Editorで実行してください。これにより、フィルタリング対応の `match_documents` 関数が作成されます。

## 📚 まとめ

### このアプリの全体像

**R&D Brain**は、化学メーカーの営業担当者が顧客との面談内容から**研究開発の新たなアイデアを創出**するためのAIエージェントです。

### 主な特徴

1. **簡単な操作**: 面談内容を入力するだけで、自動的に分析とレポート生成が実行される
2. **社内知見の活用**: ベクトル検索により、他事業部の類似知見を自動的に発見
3. **市場動向の把握**: 最新の市場トレンドを自動調査し、レポートに統合
4. **戦略的な提案**: AIが複数の情報源を統合して、実用的な新用途・改良アイデアを提案

### 技術的なポイント

- **ベクトル検索**: OpenAIのEmbedding APIでテキストをベクトル化し、意味的に類似した文書を検索
- **RAG (Retrieval-Augmented Generation)**: 社内知見と市場情報を取得し、それらを基にAIがレポートを生成
- **クロスポリネーション**: 異なる事業部間での知識の横断的な活用を促進

### 使い始めるために

1. ✅ このREADMEを最後まで読む
2. ✅ セットアップ手順に従って環境を構築
3. ✅ 実際に面談録を1件登録してみる
4. ✅ 生成されたレポートを確認し、その価値を評価する
